Intranet Local

Esta página brinda los lineamientos y reglas que utiliza este sitio, se recomienda seguirlos

Esta aplicación consiste en

  • Opciones para una Intranet, mostrar información que se conecta al sistema
  • La presentación está orientada a cualquier tamaño de pantalla (desktop, tablet, movil) usando Bootstrap
  • La autenticación está vinculada con el usuario del Active Directory
  • La autorización a opciones/acciones está integrada con del ERP de la empresa.

General

  • La aplicación está hecha en el framework MVC de Microsoft, basado en solicitudes http
  • Esto significa que todas las interacciones son por medio de acciones (metodos de un controlador)
  • Estas acciones tienen una direccion o URL (Controlador/Accion)
  • Existen acciones Regulares y acciones Ajax, solo se han usado solicitudes tipo GET y POST
  • La parte del modelo(M) es manejada por Entity Framework, la cual ejecuta Procedimientos Almacenados en su mayoría
  • La parte de vista(V) es manejada por jQuery y otros frameworks que trabajan sobre este (Bootstrap, Morris, MetisMenu)

Seguridad

  • Para autenticar el usuario se utiliza el ActiveDirectory Membership Provider, se valida con la cuenta del Dominio
  • La autorizacion está integrada con el sistema principal(Spring), siendo la Intranet la aplicación 'IN'
  • Apoyado en la seguridad del sistema principal, cada usuario carga sus opciones en el menú de acuerdo a las opciones asignadas a los grupos a los que pertenece el usuario
  • Un usuario puede pertenecer a uno o varios Grupos, es posible asignar una opción directamente a un usuario, pero esto no es recomendable
  • Las opciones(tabla:sys_menu, aplicacion:IN) se asignan a los grupos mediante el modulo de Tesorería, están amarradas por la columna 'c_nombremenu'
  • Los grupos en realidad son un usuario más con el campo c_perfil = 'GP'
  • Existe un filtro que verifica si el usuario tiene acceso a CADA acción(solicitud) que intente ejecutar
  • Para esto, la acción debe estar registrada como opción en el módulo de tesorería y asignada a los grupos pertinentes
  • Es posible saltar dicha seguridad para ciertas acciones que estén dentro de las opciones principales
  • Para saltar dicha verificación se debe colocar el Atributo [AllowLoggedUser] al método
  • Cabe resaltar que esta exclusión de autorización solo debería aplicarse para acciones Ajax, por lo tanto se recomienda agregar tambien el atributo [AjaxOnly]
  • Toda acción Ajax debería ser llamada con una solicitud POST, así que debería agregarse el atributo [HttpPost]
  • Asimismo, toda solicitud POST debe usar un Token de seguridad, por lo que se recomienda usar tambien [AntiForgeryToken]
  • El uso de los atributos mencionados se puede verificar en las acciones Account/Login o Personal/ListarDirectorioRes
